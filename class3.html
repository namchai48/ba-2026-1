ครั้งที่ 3 — Power Query: รวมและทำความสะอาดข้อมูลเซนเซอร์ฟาร์ม

คาบนี้เราจะเลิก "ก็อป–วาง–แก้มือ" ทุกเดือน แล้วเปลี่ยนเป็น สร้างขั้นตอนครั้งเดียว กดรีเฟรชได้ตลอดไป
โจทย์จริง: ไฟล์ log เซนเซอร์จาก 3 โรงเรือน คนละรูปแบบ ต้องรวมเป็นตารางเดียวให้พร้อมวิเคราะห์

สิ่งที่จะได้จากคาบนี้

เป้าหมายการเรียนรู้

1
เข้าใจ ETL
แยกออกว่างานไหนคือ Extract / Transform / Load และทำไมต้องแยกจากการวิเคราะห์
2
ทำความสะอาดข้อมูลดิบ
Remove Rows, Use First Row as Headers, Change Type, Trim, Remove Duplicates, Split Column
3

รวม log เซนเซอร์หลายโรงเรือน/หลายเดือน ให้เป็นตารางเดียว
4
Merge (คล้าย VLOOKUP)
ดึงข้อมูลแปลง/ชนิดพืช มาต่อท้าย และเข้าใจ Join แต่ละแบบ
5
อ่าน M-Code
รู้ว่าแต่ละคลิกกลายเป็นโค้ดอะไร และแก้ไขได้เมื่อจำเป็น
6
Close & Load + Refresh
ทำครั้งเดียว ใช้ซ้ำได้ทุกเดือน — หัวใจของการทำงานจริง
💡 ประโยคที่อยากให้จำ: Power Query ไม่ได้แก้ข้อมูล — มันจดจำ "ขั้นตอนการแก้" ไว้ แล้วทำซ้ำให้อัตโนมัติทุกครั้งที่มีข้อมูลใหม่

✅ ต่างจากการแก้มือใน Excel อย่างไร? แก้มือ = แก้ "ข้อมูล" (ทำครั้งเดียวใช้ครั้งเดียว) · Power Query = แก้ "สูตรของขั้นตอน" (ทำครั้งเดียวใช้ได้ตลอด)

เทียบให้เห็นภาพ: ทำมือ vs Power Query

❌ วิธีเดิม (ทำมือทุกเดือน)

  • เปิด 3 ไฟล์ ก็อปวางต่อกัน
  • ลบบรรทัดหัวรายงานทีละไฟล์
  • ไล่ลบแถวซ้ำด้วยตา
  • แยกคอลัมน์ด้วย Text to Columns
  • VLOOKUP ดึงชนิดพืช
  • เดือนหน้า → ทำใหม่ทั้งหมด (≈45 นาที)

✅ วิธี Power Query

  • Get Data → From Folder (ครั้งเดียว)
  • คลิกขั้นตอนทำความสะอาด (ครั้งเดียว)
  • Append + Merge (ครั้งเดียว)
  • Close & Load
  • วางไฟล์ใหม่ในโฟลเดอร์
  • เดือนหน้า → กด Refresh All (≈5 วินาที)

👆 ยังไม่ได้ทำอะไรเลย — สังเกตว่า 3 แถวแรกไม่ใช่ข้อมูล และหัวตารางยังเป็น Column1, Column2…

Applied Steps — สมุดบันทึกขั้นตอน

ทุกครั้งที่คลิก Power Query จะจดไว้ที่แถบขวามือ ชื่อว่า Applied Steps ลบ/แก้/สลับลำดับได้ตลอดเวลา — นี่คือสิ่งที่ทำให้ทำซ้ำได้

⚠️ ลำดับสำคัญมาก: ถ้า Use First Row as Headers ก่อน Remove Top Rows หัวตารางจะกลายเป็น "รายงานข้อมูลเซนเซอร์…" ทันที ลองสลับดูจะเห็นว่าพังทั้งคิวรี

Merge vs VLOOKUP/XLOOKUP

ประเด็นVLOOKUP / XLOOKUPMerge (Power Query)
จำนวนคอลัมน์ที่ดึงได้ทีละคอลัมน์ (ต้องเขียนหลายสูตร)เลือกหลายคอลัมน์พร้อมกัน
ข้อมูลหลายแสนแถวช้า ไฟล์บวมเร็ว ไม่เก็บสูตรในเซลล์
หาข้อมูลที่จับคู่ไม่ได้ต้องไล่หา #N/Aใช้ Left Anti Join ได้ทันที
ทำซ้ำเดือนหน้าลากสูตรใหม่กด Refresh
คีย์ซ้ำในตารางขวาได้ค่าแรกเท่านั้น (เงียบ ๆ)แถวจะเพิ่มจำนวน — ต้องระวัง!
⚠️ ข้อผิดพลาดอันดับ 1 ของนักศึกษา: ตารางขวามีรหัสแปลงซ้ำ → หลัง Merge แถวเพิ่มจาก 500 เป็น 900 โดยไม่รู้ตัว แล้วผลรวมผลผลิตเพี้ยน
เช็คเสมอ: จำนวนแถวก่อน–หลัง Merge ต้องเท่ากัน (ถ้าใช้ Left Outer และคีย์ขวาไม่ซ้ำ)
Advanced Editor

👨‍💻 ทุกคลิกของคุณ = โค้ด M

Power Query เขียนภาษา M ให้อัตโนมัติ เราไม่ต้องเขียนเองตั้งแต่ต้น แต่ ต้องอ่านออก เพื่อแก้เมื่อคิวรีพัง — เลือกขั้นตอนด้านล่างเพื่อดูโค้ดที่ถูกสร้าง

กายวิภาคของโค้ด M

let Source = Csv.Document(File.Contents("C:\Farm\sensorA.csv")), RemovedTop = Table.Skip(Source, 3), Promoted = Table.PromoteHeaders(RemovedTop), Typed = Table.TransformColumnTypes(Promoted, {{"อุณหภูมิ", type number}}), NoDup = Table.Distinct(Typed) in NoDup

เช็กลิสต์ก่อนส่งงาน Power Query

  • ชนิดข้อมูลทุกคอลัมน์ถูกต้อง (วันที่เป็น Date ไม่ใช่ Text)
  • ไม่มี Error หรือ null ค้างโดยไม่ตั้งใจ — ตรวจด้วย Column Quality
  • จำนวนแถวหลัง Merge เท่ากับก่อน Merge (ถ้าตั้งใจใช้ Left Outer)
  • ตั้งชื่อขั้นตอนและชื่อคิวรีให้สื่อความ ไม่ใช่ Query1
  • ลองลบ/เพิ่มข้อมูลในไฟล์ต้นทางแล้ว Refresh — ต้องไม่พัง
  • มีคอลัมน์ระบุแหล่งที่มา (โรงเรือน/เดือน) เสมอ
