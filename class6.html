📄 SELECT · WHERE · ORDER BY · LIMIT

SELECT คอลัมน์ที่ต้องการ -- เอาอะไรมาบ้าง FROM ชื่อตาราง -- จากตารางไหน WHERE เงื่อนไข -- เอาเฉพาะแถวไหน ORDER BY คอลัมน์ [DESC] -- เรียงอย่างไร LIMIT จำนวน -- เอากี่แถว

กดตัวอย่างด้านล่างเพื่อรันคิวรีจริงกับตาราง harvest

COUNT(*)
นับจำนวนแถว
SUM(qty_kg)
ผลรวมผลผลิต

ค่าเฉลี่ยต่อครั้ง
MIN / MAX
ค่าต่ำสุด/สูงสุด
ROUND(x, 2)
ปัดทศนิยม 2 ตำแหน่ง

นับแปลงที่ไม่ซ้ำ
SELECT plot_id, SUM(qty_kg) AS total_kg FROM harvest WHERE harvest_date >= '2026-04-01' -- กรอง "ก่อน" จัดกลุ่ม GROUP BY plot_id HAVING SUM(qty_kg) > 1000 -- กรอง "หลัง" จัดกลุ่ม ORDER BY total_kg DESC LIMIT 5;
WHERE กรองแถวดิบ · HAVING กรองผลสรุป — นี่คือคำถามข้อสอบที่ออกแทบทุกครั้ง

กฎเหล็กของ GROUP BY

-- ❌ ผิด SELECT plot_id, house, SUM(qty_kg) FROM harvest GROUP BY plot_id;
คอลัมน์ house อยู่ใน SELECT แต่ไม่อยู่ใน GROUP BY และไม่ได้ถูก aggregate → ฐานข้อมูลไม่รู้ว่าจะเอาค่าไหนของกลุ่ม
-- ✅ ถูก SELECT plot_id, house, SUM(qty_kg) FROM harvest GROUP BY plot_id, house;
ทุกคอลัมน์ใน SELECT ต้อง อยู่ใน GROUP BY หรือ ถูกห่อด้วยฟังก์ชัน aggregate อย่างใดอย่างหนึ่งเสมอ
รวมข้อมูลข้ามตาราง

🔗 JOIN — เหมือน Merge ใน Power Query

SELECT p.plot_id, p.house, c.crop_name FROM plot AS p JOIN crop AS c ON p.crop_id = c.crop_id;
ON บอกว่า "แถวไหนของสองตารางถือว่าเป็นเรื่องเดียวกัน" — มักเป็น Foreign Key = Primary Key

plot (ซ้าย)

crop (ขวา)

ผลลัพธ์

JOIN หลายตาราง — โจทย์จริงของโซ่อุปทาน

-- อัตราส่งตรงเวลาแยกตามชนิดพืช SELECT c.crop_name, COUNT(*) AS ships, SUM(CASE WHEN s.on_time = 1 THEN 1 ELSE 0 END) AS on_time_ships, ROUND(100.0 * SUM(CASE WHEN s.on_time = 1 THEN 1 ELSE 0 END) / COUNT(*), 1) AS otd_pct FROM shipment s JOIN orders o ON s.order_id = o.order_id JOIN crop c ON o.crop_id = c.crop_id WHERE s.ship_date BETWEEN '2026-04-01' AND '2026-06-30' GROUP BY c.crop_name HAVING COUNT(*) >= 10 ORDER BY otd_pct ASC;
อ่านจากล่างขึ้นบน: กรองไตรมาส 2 → รวมตาราง 3 ตาราง → จัดกลุ่มตามพืช → ตัดพืชที่ส่งน้อยกว่า 10 ครั้ง → เรียงจากแย่ที่สุดขึ้นก่อน (เพราะเราสนใจปัญหา)
💡 เทคนิค SUM(CASE WHEN … THEN 1 ELSE 0 END) คือวิธีมาตรฐานในการนับ "เฉพาะแถวที่เข้าเงื่อนไข" ใช้คำนวณ OTIF, Fill Rate, % ของเสีย ได้ทั้งหมด — จำแพตเทิร์นนี้ไว้ใช้ได้ตลอดชีวิตการทำงาน

🧪 ห้องทดลองคิวรี — ประกอบคิวรีแล้วรันจริง

เลือกส่วนประกอบด้านล่าง ระบบจะสร้างคำสั่ง SQL ให้ดู พร้อมรันกับข้อมูลจริงในตาราง harvest ที่ JOIN กับ plot และ crop แล้ว

🎯 โจทย์ท้าทาย — ลองประกอบคิวรีให้ตอบคำถามเหล่านี้

  • แปลงไหนให้ผลผลิตรวมสูงสุด 3 อันดับแรกในไตรมาส 2?
    ใบ้: GROUP BY plot_id · SUM(qty_kg) · WHERE ไตรมาส 2 · ORDER BY มาก→น้อย · LIMIT 3
  • ชนิดพืชใดมี % ของเสียสูงที่สุด?
    ใบ้: GROUP BY crop_name · % ของเสีย · ORDER BY มาก→น้อย
  • โรงเรือนไหนมีมูลค่าผลผลิตรวมมากที่สุด?
    ใบ้: GROUP BY house · SUM(qty_kg × price_per_kg)
  • เดือนไหนที่ % ของเสียเกิน 5?
    ใบ้: GROUP BY month · % ของเสีย · HAVING > 5
เข้าใจข้อนี้แล้วดีบักได้เอง

⚙️ SQL ทำงานตามลำดับไหน (ไม่ใช่ลำดับที่เราเขียน!)

ลำดับที่เราเขียน

SELECT FROM JOIN WHERE GROUP BY HAVING ORDER BY LIMIT

ลำดับที่ฐานข้อมูลทำงานจริง

1. FROM เอาตารางมาก่อน 2. JOIN รวมตาราง 3. WHERE กรองแถวดิบ 4. GROUP BY จัดกลุ่ม 5. HAVING กรองกลุ่ม 6. SELECT เลือกคอลัมน์ + ตั้ง alias 7. ORDER BY เรียง 8. LIMIT ตัดจำนวน
💡 ข้อนี้อธิบายปริศนา 2 ข้อที่นักศึกษาเจอบ่อย
1) ทำไม WHERE total_kg > 100 ถึง error? → เพราะ WHERE (ขั้น 3) ทำงาน ก่อน SELECT (ขั้น 6) ที่ตั้งชื่อ alias total_kg ตอนนั้น alias ยังไม่มีอยู่
2) ทำไม ORDER BY total_kg ถึงใช้ได้? → เพราะ ORDER BY (ขั้น 7) ทำงาน หลัง SELECT alias จึงมีแล้ว

ใช้ทักษะรวมของครึ่งเทอมแรก: Excel → Power Query → Power BI → SQL

  • เตรียมข้อมูล (Power Query) — รวมไฟล์เซนเซอร์ 3 โรงเรือน ทำความสะอาด และ Merge กับทะเบียนแปลง
  • คิวรี SQL อย่างน้อย 6 ข้อ — ต้องมี GROUP BY, HAVING, JOIN อย่างน้อยอย่างละ 1 ข้อ พร้อมอธิบายว่าแต่ละคิวรีตอบคำถามธุรกิจอะไร
  • แดชบอร์ด Power BI 1 หน้า — มี KPI การ์ด ≥ 4 ตัว (ต้องมี KPI โลจิสติกส์ ≥ 2), Time Intelligence ≥ 1 measure
  • บทวิเคราะห์ 1 หน้า — เกิดอะไรขึ้น (พร้อมมูลค่าเป็นเงิน) → ทำไม (มีหลักฐานจากข้อมูล) → เสนอให้ทำอะไร
  • นำเสนอ 7 นาที — เน้นข้อสรุปและข้อเสนอ ไม่ใช่การเดินชมเครื่องมือ
